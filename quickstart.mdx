This guide walks you through building a working echo bot — a bot that replies to every text message with the same text back. You'll have it running in about five minutes.

## Prerequisites

- Java 17 or later
- Maven 3.6 or later
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

## Build your first bot

<Steps>
  <Step title="Add the dependency">
    Add `strongram-longpolling` to your `pom.xml`. This pulls in `strongram-core` and the TelegramBots 9.x libraries automatically.

    ```xml
    <dependency>
        <groupId>ru.daniil4jk</groupId>
        <artifactId>strongram-longpolling</artifactId>
        <version>0.5.0</version>
    </dependency>
    ```
  </Step>

  <Step title="Create the bot class">
    Extend `ChainedBot` and override `getChain()` to register your handlers in order. The `ConfigurableChainFactory` lambda receives a configurator you call `.add()` on.

    ```java
    import ru.daniil4jk.strongram.core.bot.ChainedBot;
    import ru.daniil4jk.strongram.core.chainOrderConfigurator.factory.ChainFactory;
    import ru.daniil4jk.strongram.core.chainOrderConfigurator.factory.configurable.ConfigurableChainFactory;

    public class EchoBot extends ChainedBot {

        public EchoBot(String botUsername) {
            super(botUsername);
        }

        @Override
        protected ChainFactory getChain() {
            return new ConfigurableChainFactory(chainOrderConfigurator -> {
                chainOrderConfigurator.add(new EchoHandler());
            });
        }
    }
    ```
  </Step>

  <Step title="Implement the handler">
    Extend `FilteredUpstreamHandler`. Override `getFilter()` to declare when this handler should run, and `processFiltered()` to handle matching updates.

    `Filters.hasMessageText()` matches any update that contains a plain text message. `As.messageText()` extracts the message text from the context.

    ```java
    import ru.daniil4jk.strongram.core.context.request.RequestContext;
    import ru.daniil4jk.strongram.core.filter.Filter;
    import ru.daniil4jk.strongram.core.filter.Filters;
    import ru.daniil4jk.strongram.core.upstream.FilteredUpstreamHandler;
    import ru.daniil4jk.strongram.core.unboxer.As;

    public class EchoHandler extends FilteredUpstreamHandler {

        @Override
        protected Filter getFilter() {
            return Filters.hasMessageText();
        }

        @Override
        protected void processFiltered(RequestContext ctx) {
            String text = ctx.getRequest(As.messageText());
            ctx.getResponder().send("You said: " + text);
        }
    }
    ```
  </Step>

  <Step title="Start the application">
    Create a `LongPollingBotApplication`, wrap your bot in a `LongPollingBotAdapter` with your token, and register it. The adapter handles the polling loop and wires the Telegram client automatically.

    ```java
    import ru.daniil4jk.strongram.longpolling.LongPollingBotApplication;
    import ru.daniil4jk.strongram.longpolling.adapter.LongPollingBotAdapter;

    public class Main {

        public static void main(String[] args) throws Exception {
            LongPollingBotApplication application = new LongPollingBotApplication();
            application.registerBot(new LongPollingBotAdapter(
                "YOUR_BOT_TOKEN",
                new EchoBot("EchoBotUsername")
            ));
        }
    }
    ```

    Replace `YOUR_BOT_TOKEN` with your token from BotFather and `EchoBotUsername` with your bot's username (without the `@`). Run the `main` method and send a text message to your bot — it should echo it back.
  </Step>
</Steps>

<Note>
The `LongPollingBotAdapter` constructor takes the **token** as the first argument and your **bot instance** as the second. The token is used by the polling adapter; the username is passed to `ChainedBot` and is used in Telegram API calls.
</Note>

## What just happened

- `ChainedBot` builds the handler chain on first update and manages the request/response lifecycle.
- `FilteredUpstreamHandler` runs `getFilter()` on every update. If the filter matches, it calls `processFiltered()`. If not, it passes the update to the next handler in the chain.
- `ctx.getResponder().send(...)` queues the reply. Strongram sends it after the upstream pipeline finishes.
- `LongPollingBotAdapter` runs a background thread to poll Telegram for updates and dispatch them to your bot.
