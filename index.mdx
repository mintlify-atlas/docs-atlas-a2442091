Strongram is a modular Java framework built on top of the TelegramBots library that lets you build production-ready Telegram bots with clean, structured code. Instead of dealing with raw `Update` objects and manual dispatch logic, Strongram gives you a composable chain of handlers, a powerful filter system, multi-step dialog flows, interactive keyboards, and a one-line message API — all wired together with a minimal setup.

## Key features

<AccordionGroup>
  <Accordion title="Chain-of-responsibility handler pipeline">
    Every incoming Telegram update flows through an ordered list of `UpstreamHandler` instances. Each handler either processes the update or passes it to the next handler in the chain. You control the exact order — add exception reporters first, domain handlers in the middle, and a fallback handler last.
  </Accordion>
  <Accordion title="40+ declarative filters">
    `Filters.hasMessageText()`, `Filters.isPrivateChat()`, `Filters.textContains("buy")`, and 40 more static factories. Combine them with `.and()`, `.or()`, and `.negate()` to express any routing condition without boilerplate `if` trees.
  </Accordion>
  <Accordion title="State-machine dialogs">
    `DialogImpl` maps enum values to `DialogPart` steps. Each step declares its filter (when it can accept input), a notification message (sent when the step becomes active), and a handler that can advance state, read scoped storage, and stop the dialog.
  </Accordion>
  <Accordion title="Interactive keyboards">
    `InlineInteractiveButton` and `ReplyInteractiveButton` carry their own `Consumer<RequestContext>` action. Register an `InteractiveKeyboardUpstreamHandler` and Strongram routes button presses to the right lambda automatically.
  </Accordion>
  <Accordion title="Modular transport">
    Add `strongram-longpolling` for development and simple deployments, or `strongram-webhook` for production. Both share the same `ChainedBot` core — switch transports without changing your business logic.
  </Accordion>
</AccordionGroup>

## Getting started

<Steps>
  <Step title="Add the dependency">
    Add `strongram-longpolling` or `strongram-webhook` to your `pom.xml`. See [Installation](/installation) for the full Maven coordinates.
  </Step>
  <Step title="Create your bot class">
    Extend `ChainedBot` and override `configureUpstream()` to register your handlers in order.
  </Step>
  <Step title="Implement handlers">
    Extend `CommandUpstreamHandler`, `FilteredUpstreamHandler`, or `BaseUpstreamHandler` to handle messages, commands, and callbacks.
  </Step>
  <Step title="Start the application">
    Wrap your bot in a `LongPollingBotAdapter` and register it with `LongPollingBotApplication` to start receiving updates.
  </Step>
</Steps>
